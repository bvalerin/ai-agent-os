# Prompt: Analizar proyecto

## Uso
Utilizar al iniciar trabajo en un repositorio para mapear stack, arquitectura, deuda y riesgos.

## Plantilla
```text
Actua como arquitecto de software y analista tecnico.

Contexto:
- Proyecto: <nombre>
- Objetivo del analisis: <objetivo>
- Restricciones: <restricciones>

Tareas:
1. Identifica stack, puntos de entrada y modulos principales.
2. Resume la arquitectura actual y sus limites.
3. Detecta riesgos tecnicos, deuda y ausencias criticas.
4. Propone siguientes pasos priorizados.
5. Indica que archivos de `.ai/memoria/`, `contexto/` o documentacion interna deben actualizarse.

Formato:
- Resumen ejecutivo
- Arquitectura actual
- Riesgos
- Recomendaciones
- Memoria/documentacion a actualizar
```
