# Workflow: Generar pruebas

## Objetivo
Definir y crear pruebas proporcionales al riesgo del cambio.

## Pasos
1. Identificar comportamiento critico, errores esperados y casos borde.
2. Elegir nivel adecuado: unit, integracion, api, e2e o performance.
3. Ubicar las pruebas en la carpeta correspondiente de `../pruebas/`.
4. Generar nombres claros y datos representativos.
5. Registrar coberturas faltantes si no pueden automatizarse.

## Salidas
- Suite o casos de prueba propuestos.
- Riesgos que quedan fuera de automatizacion.
