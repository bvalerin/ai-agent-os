# Workflow: Investigar bug

## Objetivo
Reproducir, aislar y documentar un defecto hasta dejarlo listo para correccion y regresion.

## Pasos
1. Reproducir con pasos minimos y consistentes.
2. Capturar entorno, datos, logs y comportamiento observado.
3. Delimitar capa probable: frontend, backend, datos o infraestructura.
4. Medir severidad, impacto y frecuencia.
5. Proponer prueba de regresion.
6. Registrar hallazgos y workaround si existe.

## Salidas
- Bug reproducible.
- Hipotesis de causa.
- Caso de regresion propuesto.
