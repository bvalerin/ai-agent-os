# Dominio

## Problema que resuelve
Los proyectos con agentes suelen degradarse cuando no existe una forma consistente de coordinar roles, documentar decisiones y preservar contexto.

## Conceptos clave
- Agente: rol especializado con reglas, entradas y salidas.
- Workflow: procedimiento repetible para una tarea.
- Memoria: conocimiento persistente del proyecto.
- Prompt: instruccion reutilizable para orientar analisis o generacion.
- Contexto: informacion base del stack, entornos e integraciones.
- MCP: mecanismo para conectar herramientas de trabajo a los agentes.

## Objetivos del dominio
- Estandarizar colaboracion humano-agente.
- Reducir improvisacion en cambios tecnicos.
- Hacer visible el riesgo residual de cada cambio.
- Crear una base auditable para evolucionar el proyecto.
